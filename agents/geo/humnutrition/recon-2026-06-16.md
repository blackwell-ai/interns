# HUM Nutrition recon and frozen truth table, June 16, 2026

Internal working notes for the AI visibility audit of humnutrition.com. Methodology:
`brain/company/audit-methodology.md`. Passive recon and the truth-table freeze ran
first, before any engine query, per the charter. Live engine testing (the two-pass
prompt battery) is the next phase and still open.

All checks run June 16, 2026 from curl with named user-agents. Site sits behind
Cloudflare. Platform is Astro (asset paths under `/_astro/`), custom storefront,
not Shopify (`/collections/all` 404s, no myshopify or cdn.shopify signals).

## Headline: this is the inverse of Good Molecules

Good Molecules blocked the AI assistants at the WAF and shipped malformed,
JavaScript-only schema. HUM does the opposite. It serves full, well-formed Product
schema in raw HTML to the AI crawlers, and the one crawler it blocks is Bing's.

| **Bingbot** | **403** | **blocked at Cloudflare edge** |

Bingbot detail: deterministic 403 across three retries, on every path tested
(`/`, `/products`, `/sitemap.xml`), and on both apex and www. The 403 is served by
Cloudflare (`server: cloudflare`, `cf-ray` present), so the block sits at the edge,
not in robots.txt. robots.txt names only Googlebot and Googlebot-image as allowed,
then a `User-Agent: *` group that disallows only funnel and account paths
(`/checkout`, `/my_account`, `/recommendations`, `/free_trial/*`, `/survey/t`). It
carries no AI-crawler rules at all, so the AI assistants are allowed by default and
the Bing block is a deliberate WAF action, not a robots directive.

Why it matters: ChatGPT, Claude, and Perplexity can read humnutrition.com directly.
Anything that leans on the Bing index, namely Microsoft Copilot and parts of Bing's
own answer surface, cannot index the site. This is the one closed channel, and it is
the mirror image of the usual finding. Confirm the downstream effect in live testing.

## Structured data (PDP: /product/15/collagen-love, server-rendered, raw HTML)

Three JSON-LD blocks in source, all parse, all canonical:

- Product, `@context: https://schema.org`, `@type: Product` (correct casing and https
  context, the two things Good Molecules got wrong). Carries brand, description, sku.
- Offer nested in Product: price 40.00, priceCurrency USD, priceValidUntil 2027-06-15,
  availability InStock, itemCondition NewCondition, hasMerchantReturnPolicy.
- AggregateRating: 4.53 over 198 reviews. Ten Review and Person nodes embedded.
- Separate BreadcrumbList and FAQPage blocks.
- OpenGraph product meta also present in source: `product:price:amount` 40.00,
  `product:price:currency` USD, `product:brand`, `product:condition` new,
  `product:retailer_item_id` 15.

The schema is a strength, not a finding. It is server-rendered, so a crawler that does
not run JavaScript still reads price, stock, rating, and returns policy.

absence is authoritative. ACP and AP2 have no public well-known path to probe (they run
off a registered product feed and a checkout endpoint), so curl cannot positively confirm
them; none of the probeable variants exist, and there is no first-party checkout surface.

Positive surfaces reconfirmed via curl the same pass: robots.txt carries no AI-crawler
directives; the PDP serves three canonical JSON-LD blocks (Product with Offer price 40.00
/ USD / InStock / priceValidUntil 2027-06-15 / NewCondition / hasMerchantReturnPolicy,
AggregateRating 4.53/198, BreadcrumbList, FAQPage); OpenGraph and product meta present in
source; AI crawlers 200 and Bingbot 403, unchanged.

## Frozen truth table

Full catalog pulled from PDP schema, saved to `truth-table-raw.csv` (37 products).
Prices run $12 to $60. Ratings cluster 4.2 to 4.75. Review depth is real: Flatter Me
2,869, Daily Cleanse 2,493, Private Party 1,892, Hair Strong Gummies 1,219, Gut
Instinct 1,207, Hormone Balance 1,193. This is a well-reviewed catalog, which is
material for the Reputation and Recommendability dimensions.

Naming note: several PDPs serve a rebranded name under an old slug. `/moody-bird`
renders "Hormone Balance", `/hair-sweet-hair` renders "Hair Strong Gummies",
`/glow-sweet-glow` renders "Hyaluronic Glow Gummies", `/calm-sweet-calm` renders
"Ashwagandha Calm Gummies". An engine that learned the old name from third-party
sources may not match it to the current page. Worth a live check.
